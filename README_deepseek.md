# Docs Assembler

**A VS Code extension that treats documentation like code.**

[Live Demo](https://netoftrees.com/docs-assembler-demo/)  
[Install from Marketplace](https://marketplace.visualstudio.com/items?itemName=netoftrees.docs-assembler)  
[GitHub](https://github.com/netoftrees/docs-assembler)



## Stop documenting in straight lines

Human knowledge branches, loops, and layers – a living web of what-ifs and dependencies. Traditional documents force this into linear prose, losing the very structure that makes expertise usable.

Docs Assembler bridges that gap. It gives form to the networks in your mind – for others to follow, learn from, or extend with their own thinking.

---

## Build documentation like Lego

Capture complex thinking. Simplify intricate systems. Inspired by code classes.

**Maps (.tsmap files)** – Self-contained documentation modules that can be nested and composed, just like classes.

**Variables** – Define reusable text snippets. Change one, update everywhere.

**Inheritance & Composition** – Build complex guides from simple, reusable blocks.

**Compile to Docs** – Assemble components on-the-fly into flawless, context-aware Markdown or HTML.

---

## Designed for developers

| Principle | What it means |
|-----------|----------------|
| **Absolute Ownership** | Your docs live as Markdown and JSON in your Git repo. No subscription. No SaaS lock-in. |
| **Zero Lock-In** | Publish to standard Markdown. Uninstall the extension and your docs are still perfectly usable. |
| **GitHub Pages Native** | Publish directly to GitHub Pages. Liquid scripts work as expected. |
| **Engineering Rigour** | IntelliSense for variables, full validation before publish, automatic relative URL handling. |

## Get started

1. **See it in action** – [Live Demo](https://netoftrees.com/docs-assembler-demo/) (requires JavaScript)
2. **Explore the demo repo** – [github.com/netoftrees/docs-assembler-demo](https://github.com/netoftrees/docs-assembler-demo)
3. **Clone the template** – [github.com/netoftrees/docs-assembler-template](https://github.com/netoftrees/docs-assembler-template) (includes Jekyll setup for local preview)
4. **Install the extension** – [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=netoftrees.docs-assembler) | [Open VSX](https://open-vsx.org/extension/netoftrees/docs-assembler)

**After an extension update:** Open Command Palette (`Cmd+Shift+P`) → *Clear Editor History* – this refreshes map views.
