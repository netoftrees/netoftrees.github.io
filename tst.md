# Docs Assembler
#### Build documentation like Lego

[Live Demo](https://netoftrees.com/docs-assembler-demo/)  
[Install](https://open-vsx.org/extension/netoftrees/documentation-assembler)  
[GitHub](https://github.com/netoftrees/docs-assembler)

Knowledge is a living web of what-ifs, dependencies, and context. That changes.

If you think of knowledge like code, you can solve documentation the same way:

- **A text editor** you already know  
- **Classes and variables** - a single source of truth, zero duplication  
- **IntelliSense** that corrects as you type  
- **A compiler** that validates, handles relative URLs automatically, and assembles the final output

The classes in this case are a type of decision tree called **maps**, so they inherently have branching when you need it.

**Docs Assembler** is a VS Code extension that does exactly that.  
Capture complex thinking. Simplify intricate systems.

- **Absolute Ownership** - Your docs live as Markdown and JSON in your Git repo. No subscriptions.
- **Zero Lock-In** - Publish to standard Markdown. Uninstall the extension and your documentation can be edited with any text editor.
- **GitHub Pages** - Publish directly to GitHub Pages. Liquid scripts work as expected.

The result: your documentation lives in Git repos with full change control, and publishes to GitHub Pages for free.
